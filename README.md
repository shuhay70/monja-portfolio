## プロジェクト名

もんじゃ専門店向け デジタルメニュー／注文補助アプリ

## アプリの概要

来店客（特に外国人観光客）がQRコードからアクセスし、言語を選んでメニュー閲覧・アンケート回答・注文確認まで完結できるアプリです。店舗スタッフの説明負荷を減らし、言語の壁による注文の不便さを解消することを目的に設計しました。

## 主な機能一覧

- QRコード経由でのみ閲覧可能なアクセス制御
- 4言語対応（日本語／英語／中国語／韓国語）のメニュー表示
- カテゴリ切替・詳細説明・価格・画像付きメニュー一覧
- もんじゃの歴史や焼き方ガイドの閲覧
- 来店アンケート（人数／出身国／来店きっかけ／初回かどうか）
- 管理画面でアンケート結果を一覧表示

## こだわった技術的ポイント

- **QR起点のアクセス制御設計**  
  短時間で失効するトークンを発行し、閲覧の入口をQRに限定。URL共有やブックマークからの不正閲覧を防ぐため、トークンはCookieに保存し、サーバー側の検証を必須にしています。
- **期限切れの即時検知**  
  ミドルウェアによる検証に加え、フロント側でも有効期限を監視し、ページ遷移がなくても失効を検知して案内画面へ遷移させています。
- **多言語対応の設計**  
  ルーティングと翻訳メッセージを分離し、言語追加や文言修正が容易になるよう構成。メニューや説明文の管理負荷を下げています。
- **管理機能**  
  アンケートのデータを保存し、管理画面で参照できるようにすることで、店舗改善につながる情報活用を意識しています。

## 使用技術スタック

- **言語**: TypeScript
- **フレームワーク**: Next.js（App Router）, React
- **i18n**: next-intl
- **UI**: Tailwind CSS, React Icons
- **バックエンド／DB**: Supabase（PostgreSQL）
- **セキュリティ**: JWT（`jose`）によるトークン検証
- **インフラ**: Vercel想定
